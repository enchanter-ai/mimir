# Mimir On-Chain Anchor

The settlement layer for Mimir Provenance Envelopes. Each off-chain envelope is anchored on-chain by its keccak256 digest, so a third party can later challenge it and (in the EigenLayer AVS slice) slash the issuer's restaked stake.

## What's in here

| Path | Purpose |
|---|---|
| `contracts/MimirValidationRegistry.sol` | ERC-8004 Validation Registry contract. `register / registerBatch / revoke / verify / isValid / exists`. Solidity ^0.8.20. Zero external deps. |
| `contracts/MimirValidationRegistry.t.sol` | Foundry-style test contract (optional — only runs if Foundry installed). |
| `compile.js` | One-shot solc-js compile script. Reads the `.sol`, emits ABI + creation bytecode to `go/abi/`. Run after editing the contract. |
| `go/anchor.go` | Go client. Talks to any EVM-compatible RPC endpoint. Embeds the compiled bytecode for in-process deployment. |
| `go/anchor_test.go` | Full integration suite running against go-ethereum's in-process simulated backend — **no external tooling required**. |
| `go/abi/MimirValidationRegistry.json` | Compiled ABI (regenerated by `compile.js`). |
| `go/abi/MimirValidationRegistry.bin` | Compiled creation bytecode (regenerated by `compile.js`). |

## Run the tests (zero external tooling)

```bash
cd anchor/go
CGO_ENABLED=0 go test -v ./...
```

That alone proves: contract compiles, deploys, register + verify round-trips, duplicate registrations revert, revocation flips the flag, expiry causes `isValid → false`, re-revoke reverts, and a 5-envelope lifecycle with partial revocation behaves correctly.

The simulated backend is a pure-Go EVM provided by `github.com/ethereum/go-ethereum/ethclient/simulated`. It is bit-identical to mainnet EVM semantics for our contract's needs.

## Regenerate bytecode after editing the contract

```bash
cd anchor
node compile.js
```

(Requires `npm install --no-save solc@0.8.20` once.)

## Deploy to a real network

```bash
# Sepolia / Holesky / mainnet
export RPC_URL=https://sepolia.infura.io/v3/<key>
export DEPLOYER_KEY=<hex-private-key-of-funded-wallet>

# Example with cast (Foundry); or use the embedded bytecode via a custom Go runner.
cast send --create $(cat go/abi/MimirValidationRegistry.bin) \
  --rpc-url $RPC_URL --private-key $DEPLOYER_KEY
```

After deployment, anchor calls go through the `anchor.Client`:

```go
c, _ := anchor.New(rpcURL, privKeyHex, deployedAddr)
txHash, _ := c.AnchorEnvelope(ctx, envelopeDigest, expiry)
receipt, _ := c.WaitMined(ctx, txHash, 30*time.Second)
```

## EigenLayer slashing — wired

As of 2026-05-15, the contract is **wired for slashing**:

1. **`register()`** is gated by `IServiceManager.isOperator(msg.sender)`. Non-operators get `NotAnOperator(caller)`. The `issuer` parameter must equal `msg.sender` (anti-spoofing — you can only anchor your own envelopes).
2. **`revoke()`** calls `ISlasher.slash(issuer, slashWad, keccak256(proof))` on every accepted fraud proof. The slash amount is set at construction time via `slashWad` (defaults to `1e17` = 10% of allocation per slash; pass any non-zero WAD to override). A `SlashTriggered` event is emitted alongside the existing `Revoked` event.
3. **Mode selection.** Pass `(address(0), address(0), 0)` to the constructor for permissionless dev mode (no operator gating, no slashing). Pass non-zero `(manager, slasher, wad)` for AVS mode.

The minimal EigenLayer interface surface — `IServiceManager.isOperator` and `ISlasher.slash` + `totalSlashed` — lives in [`contracts/IEigenLayer.sol`](contracts/IEigenLayer.sol). Test mocks ship in `MockServiceManager.sol` + `MockSlasher.sol`; the full Go test suite in `go/eigenlayer_test.go` exercises:

| Test | What it proves |
|---|---|
| `TestAVSRegisterRequiresOperator` | `register()` reverts for non-operators with `NotAnOperator` |
| `TestAVSRegisteredOperatorCanAnchor` | registered operator can anchor |
| `TestAVSRevokeTriggersSlash` | revoke → `Slasher.slash()` with the configured `slashWad`; `totalSlashed` accumulates |
| `TestAVSSlashIsolatedPerOperator` | slashing one operator does not touch another's stake |
| `TestAVSRegisterRejectsForeignIssuer` | cannot anchor on behalf of someone else (`IssuerMustBeCaller`) |

All 12 anchor tests (7 permissionless + 5 AVS) pass against the simulated EVM in under 3 seconds.

### Deploying with real EigenLayer

For Holesky / mainnet, replace the mock addresses with EigenLayer core deployments. As of 2026-Q1, the canonical mapping is:

| Mimir constructor param | Real EigenLayer contract |
|---|---|
| `IServiceManager` | the deployed AVS `ServiceManagerBase` subclass — operator opt-in is bridged to the core `DelegationManager` here |
| `ISlasher` | EigenLayer v2 `AllocationManager` — the `slash()` entry point that reduces operator magnitude |

Operator step (Holesky):

```bash
# 1. Deploy MimirValidationRegistry pointing at Holesky EigenLayer core.
export RPC_URL=https://ethereum-holesky.publicnode.com
export DEPLOYER_KEY=<funded-wallet-hex>
export SERVICE_MANAGER=<AVS ServiceManager addr on Holesky>
export ALLOCATION_MANAGER=<EigenLayer AllocationManager addr on Holesky>
export SLASH_WAD=100000000000000000   # 10%

# 2. Build constructor calldata (Go helper, no Foundry needed):
go run ./scripts/deploy.go \
  --service-manager $SERVICE_MANAGER \
  --slasher $ALLOCATION_MANAGER \
  --slash-wad $SLASH_WAD \
  --rpc $RPC_URL \
  --key $DEPLOYER_KEY
```

The `scripts/deploy.go` helper is a 50-line script that ABI-encodes the constructor args, signs a contract creation transaction, and prints the deployed registry address. It's the production-equivalent of `deployRegistry()` in `go/deploy_helpers_test.go`. Not yet written — operator step.

### What this DOES prove

A real EigenLayer integration uses the same `slash()` ABI; passing this test suite against `AllocationManager` on Holesky requires:
- a registered AVS with `ServiceManagerBase` deployed
- operator(s) with allocated stake delegated to the AVS
- the `MimirValidationRegistry` granted the AVS slashing permission

That's an operator workflow, not a code workflow. The wiring code is now proven.

### What this does NOT prove

- A real `AllocationManager.slash()` call against Holesky / mainnet. The mock matches the interface signature, not the exact gas / authorization / event semantics of the live contract.
- That EigenLayer's slashing model meets Mimir's specific economic-security requirements for "trust-anchored" envelopes (that's a tokenomics question, not a code question).
- That the slashWad is correctly calibrated (10% per slash is the default; the right number depends on operator-stake distribution and the cost-of-fraud target).
