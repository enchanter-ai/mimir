## Attack 11 — Extra JSON whitespace added (should VERIFY, not reject)

The envelope has been pretty-printed with 4-space indentation instead of the compact form. No field values have been changed; only serialization whitespace differs. A correct verifier **MUST** accept this and return VERIFY, because the verification path parses the JSON first, then strips `signature.value`, then applies RFC 8785 JCS (which produces a unique canonical form regardless of input whitespace). The canonical bytes produced by the verifier are identical whether the input was compact or pretty-printed. If the verifier rejects this it has a bug: it is comparing raw bytes instead of canonical form. Spec §9.2 (Compute Canonical Form) mandates JCS canonicalization per RFC 8785; §15.16 (Forward-Compatibility) requires unknown-field and whitespace tolerance.
